## 目的

- 今回のタスクに関連する既存システムの仕組みを把握する
- 今回のタスクに関連する技術・概念・前例を調査する
- 後続の要求定義、要件定義、対応方針決めに必要な判断材料を揃える

このスキルでは調査結果の整理までを行います。
対応方針の決定や詳細設計は行わないでください。

## 手順

1. まず、以下の候補をもとに、今回の調査をどのまとめる際の視点をいくつか選んでください
  - 候補
    - 既存システムの概要（原則として含めてほしいです）
    - 動作フロー
    - データ構造・テーブル構成
    - 関連技術
    - 技術的な制約
    - 類似実装・前例・参考事例
2. 各視点ごとに、必要なコード、既存ドキュメント、チケット、参考資料を調査してください
3. 各視点ごとに、調査結果を小見出し内へ整理してください
  - 事実・推測・未確認事項が混ざらないようにしてください
  - 推測が入る場合は、その根拠と推測であることを明記してください
4. 調査の途中で新しい論点が見つかった場合は、必要に応じて視点を追加・修正してください

## 出力フォーマット

```
## 調査結果

### {小見出し}
- {調査結果}
  - {必要に応じて補足}
  - ...
- ...
```

## 注意点

- 対応方針決めなどはこのスキルでは行いません。出力しないでください
- 調査結果と意思決定を混ぜないでください
- 事実に基づく説明を優先し、判断が必要な点は未解決事項として補足的に明記してください
