# 実装 (implement)

このスキルは、実装計画のステップ（粒度）ごとにコミットを積みながら、着実に機能を実装するためのガイドラインです。

## 指針

- **計画の粒度でコミット**:
  - 原則として、実装計画の1つのステップ（作業単位）につき1つのコミットを作成してください。
  - 実装計画に記載された「期待される成果」が達成されたタイミングでコミットを行います。
- **検証後のコミット**:
  - 各ステップの実装完了後、必ずテストや動作確認を行い、正常に動作することを確認してからコミットしてください。
- **メッセージの整合性**:
  - コミットメッセージは、実装計画のステップ内容と整合性が取れるように記述してください。

## 基本的な流れ

1. **計画の確認と準備**:
   - 実装計画（作業ステップと期待される成果のリスト）が既に存在するか確認してください。
   - **計画がない場合**: 自律的に実装計画（ステップごとの作業内容とコミット単位）を策定してください。
2. 計画の1ステップ分を実装する
3. テストや動作確認で動作を検証する
4. `git add` と `git commit` を実行し、そのステップの完了を記録する
5. 全てのステップが完了するまで2〜4を繰り返す

## 進め方の例

以下は、ある実装計画があったとき、その実装をスキル通りに行う例です

実装計画例:
```
## 実装計画

- コードの骨組み作成
  - 追加: `app/services/user_bio_updater.rb`
    - クラス定義と `#call` メソッドのシグネチャのみ
  - コミット: `UserBioUpdaterの骨組みを作成`

- テストの作成
  - 追加: `spec/services/user_bio_updater_spec.rb`
    - 正常に更新されること、および文字数制限のテスト構造を追加
  - コミット: `UserBioUpdaterのテストを実装`

- 中身の実装
  - 修正: `app/services/user_bio_updater.rb`
    - `#call` メソッドに更新ロジックとバリデーションを追加
  - コミット: `UserBioUpdaterの処理を実装`

- lintやフォーマットの修正
  - 修正: 各ファイル
  - コミット: `コードのlintとフォーマットを修正`
```

作業例：
1. 「コードの骨組み作成」を実装し、シンタックス確認後に `UserBioUpdaterの骨組みを作成` でコミット
2. 「テストの作成」を実装し、失敗することを確認後に `UserBioUpdaterのテストを実装` でコミット
3. 「中身の実装」を実装し、テスト通過を確認後に `UserBioUpdaterの処理を実装` でコミット
4. 「lintやフォーマットの修正」を実行し、`コードのlintとフォーマットを修正` でコミット
