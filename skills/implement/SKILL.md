# 実装 (implement)

このスキルは、実装計画のステップ（粒度）ごとにコミットを積みながら、着実に機能を実装するためのガイドラインです。

## 指針

- **計画の粒度でコミット**:
  - 原則として、実装計画の1つのステップ（作業単位）につき1つのコミットを作成してください。
  - 実装計画に記載された「期待される成果」が達成されたタイミングでコミットを行います。
- **検証後のコミット**:
  - 各ステップの実装完了後、必ずテストや動作確認を行い、正常に動作することを確認してからコミットしてください。
- **メッセージの整合性**:
  - コミットメッセージは、実装計画のステップ内容と整合性が取れるように記述してください。

## 基本的な流れ

1. **計画の確認と準備**:
   - 実装計画（作業ステップと期待される成果のリスト）が既に存在するか確認してください。
   - **計画がない場合**: 実装を開始する前に、自律的に実装計画（ステップごとの作業内容とコミット単位）を策定してください。ここで作成した実装計画についてユーザーの承認を得る必要はありません。
2. 計画の1ステップ分を実装する
3. テストや動作確認で動作を検証する
4. `git add` と `git commit` を実行し、そのステップの完了を記録する
5. 全てのステップが完了するまで2〜4を繰り返す
